// 1 - Acessar https://github.com
// 2 - Ir em Sign Up para criar uma conta
// 3 - Passe seu email, uma senha e um nome de usuário desejado.
// 4 - Faça o restante dos procedimentos como verificar se não é um Robô e verificação de E-mail.

// 5 - Pronto, seu perfil vai logar e agora podemos criar nosso primeiro "REPOSITÓRIO"
// É como se fosse uma pasta online do google drive que armazenamos o projeto
// 6 - No lado esquerdo aparece a opção "Criar primeiro repositório"
// 7 - Nome definimos como primeiro-repositorio sem colocar o acento.
// 8 - Na descrição você pode botar nada ou qualquer coisa, de preferência "É meu primeiro projeto"
// 9 - Será público, então todos poderão ver.
// 10 - Você deve escolher a opção Desktop no canto superior esquerdo, vamos baixar interface que facilita o desenvolvimento.
// 11 - No novo link baixar e começar a instalar o github Desktop.


// 12 - Na instalação uma das primeiras opções é "Configurar usuário e email" você vai selecionar exatamente seu email atual.
// 13 - Continuando a configuração, você vai cair em uma tela chamada "Let's get started!" ou "Vamos começar".
// 14 - Na aba "Seus repositórios" vamos selecionar o "primeiro-repositorio" e Clonar ele.
// 15 - Troque o local "Local path" para área de trabalho, vamos clonar o repositório lá.

// 16 - Com o repositório na área de trabalho nós vamos abrir ele com o visual studio code
// 17 - Você vai começar a fazer seu primeiro commit no repositório
// 18 - Crie um arquivo de texto dentro de primeiro-repositorio chamado "arquivo_do_meu_primeiro_commit"

// 19 - Dentro do github desktop vai aparecer o arquivo "arquivo_do_meu_primeiro_commit" para você marcar
// 20 - Depois de marca-lo na caixa abaixo escreve o nome do commit assim "meu primeiro commit"
// 21 - Agora você pode ir em "Commit to main" para tentar commitar seu commit.
// 22 - Como não tem nenhuma branch criada ainda, na direita ele vai perguntar se você deseja criar a clássica e primeira "branch main" ou seja "galho principal"

// 23 - Seu commit foi enviado para a branch "main" e se você visitar o repositório online vai ver seu arquivo lá
// Obs: Se você for no "primeiro-repositorio", "insights" e depois "Network" vai poder ver todas as branchs e commits feitos, na caso vai ser só um pontinho pelo fato que fizemos um commit em uma única branch chamada "main".
// 24 - Vamos fazer um segundo commit chamado "segundo commit". nisso você vai criar um arquivo chamado "arquivo_do_segundo_commit".
// 25 - Atualize o repositório web "primeiro-commit" e veja o resultado.
// Obs: Se você for no "primeiro-repositorio", "insights" e depois "Network" vai poder ver todas as branchs e commits feitos. Agora tem dois commits na branch "main"

// 26 - Agora crie um repositorio chamado "meu-segundo-repositorio" e faça o commit de um arquivo chamado "primeiro_commit_do_segundo_repositorio.js".
// Obs: O botão para criar o repositório está agora dentro do seu perfil em "repositories", você pode usar o botão "new" ou "criar"
